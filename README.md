# Fuck 60Hz (去你大爷的 60Hz)

一个非常轻量的 Magisk / KernelSU / APatch 全局高刷守护模块。

没别的功能，就是为了干翻 HyperOS (澎湃OS) 脑残的温控锁频和电量回弹，把刷新率死死钉在 120Hz。

## 🤬 为什么要做这个？
用过澎湃的都懂，官方的锁频策略极其恶心。
日常用着好好的，只要手机稍微热一点，或者电量低一点，甚至只是打开个输入法、看个视频，系统就会偷偷摸摸把刷新率降到 60Hz，瞬间卡得像 PPT。

试过市面上很多工具，要么得装一堆后台常驻软件，要么操作太复杂。索性自己用哈基米（Gemini）手搓了一个最纯粹的模块，开机自动守护，不吃内存，完全不影响系统自带的游戏空间和狂暴引擎性能调度。

## 测试环境

设备：
- Xiaomi 12S

系统：
- HyperOS 3.0.303.0（ProjectEthanol 1.3.5）

## ✨ 核心逻辑
1. **防呆硬件检测**：刷入时自动读取屏幕的物理硬件参数，如果检测到设备压根不支持 120Hz，会自动终止安装，防止小白盲目刷入。
2. **底层属性修正**：利用 `resetprop` 强行把系统硬编码里默认的 60Hz 限制直接重写改成 120Hz。
3. **常驻死守**：开机 30 秒后自动在后台每 5 秒重写一次系统的 `peak_refresh_rate` 数据库，绝不给温控任何反弹的机会。

## 📦 安装方法
1. 在Code - Download ZIP 下载最新版的 `fuck_60hz-main.zip`
3. 丢进 Magisk / KernelSU / APatch 刷入
4. 重启手机

## ⚠️ 避坑说明
- 因为本模块会接管系统的温控锁频，一些场景（如边充边玩、夏天重度使用等）发热可能会比以前明显
- 刷入后，不要去系统自带的“显示”设置里来回手动切刷新率，否则会失效
> [!TIP]
> 注：该模块可能需要Moka等模块的搭配
